The COSMOS Diagnostic tool comes in many parts. The 1st is the complete diagnostic (Form seen above) that will run through all the other diagnostics without the user needing to run each tool seperately. The diagnostic report that is created can be saved to a file that could then be sent to Velmex Technical Support if requested.



The diagnostic of the processor reports the Version, date and mode of the VXM



The diagnostic of the operating mode reports the settings involving each of the items seen to the right. (See VXM Users Manual for details)



The diagnostic of the programs reports back items such as the current program number, the current program associate (see VXM Users Manual for details), and the programs currently stored in each program number.



The diagnostic of the Axis repors back the current motor and limit settings for each axis

Typical Motor Values




Typical Limit Values



The diagnostic of the analog joystick reports back items such as the Deadband, Primary and Secondary speed ranges.



The diagnostic of the digital joystick reports back items such as the Primary and Secondary speed ranges.



The diagnostic of the output pulse reports the duration of the pulse-width and the setting for “Pulse every N steps” (see VXM users manual for details) for each axis



The diagnostic of the user outputs reports the state of the user outputs from the VXM (master VXM only)



The diagnostic of the user inputs reports the state of each of the user inputs and each of the jog buttons on the VXM



The diagnostic of the multifunction user inputs reports

Capture Motor Position on Input 4 trigger


Program # Select with Inputs 2 and 3


Jump to Program 4 after Stop input


Decel/Hard stop settings


Low valid time for Run and Input 1


Input 3 Interrupt User Waits and Jump to Program 3 enable/disable


Stop (Input 4) enable/disable


Run enable/disable



The diagnostic of the analog input reports the value of the analog input potentiometer. (Value will be between 0 and 1024)



The diagnostic of the overtravel and backlash settings reports if enabled and if so then the number of steps used for backlash compensation



The diagnostic of the limits reports the state of each limit on each axis
